🚀 DevOps Portfolio Project Guide

This guide provides an overview of the tools and commands used to build and deploy a simple Node.js application using GitHub Codespaces, Docker, and GitHub Actions. It is designed for beginners learning DevOps practices.

🌐 Why GitHub and Codespaces?

  • GitHub: Provides version control, remote repository, GitHub Actions (for CI/CD), and public sharing.
  • Codespaces: Offers a browser-based development environment with terminal access — no need to install anything locally.

🐳 Why Docker?

Docker allows packaging your application with all its dependencies into a container. This ensures it runs the same way in any environment — perfect for DevOps!

⚙️ Docker Commands Used

# Build Docker image
docker build -t devops-portfolio .

# Run Docker container
docker run -p 3000:3000 devops-portfolio

# View running containers
docker ps

# Stop container (from terminal, press Ctrl + C or use docker stop)

✅ GitHub Actions - CI/CD Pipeline

We created a workflow file .github/workflows/ci.yml to automatically run tests when code is pushed. This mimics Continuous Integration (CI) in real-world projects.

📦 Deployment (Locally)

  1. Build Docker image: docker build -t devops-portfolio .
  2. Run it locally: docker run -p 3000:3000 devops-portfolio
  3. Open browser at http://localhost:3000

📘 Summary

  • Used GitHub for code hosting and GitHub Actions for CI.
  • Used Docker for consistent environment setup.
  • Used Codespaces for a quick cloud development setup.
  • Added a badge to showcase build status.

✨ This project is beginner-friendly and can be added to your portfolio to show understanding of real-world DevOps workflows.
